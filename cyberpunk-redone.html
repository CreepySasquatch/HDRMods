💡 Tip — Display Requirements

Make sure your display actually supports HDR before installing RenoDX. Your monitor should be rated HDR400 or higher — at least 400 nits peak brightness.

Don't confuse this with TB400 or higher. TB stands for True Black. True Black monitors offer a separate True Black mode with lower peak brightness but better blacks. Set RenoDX's peak brightness to match whichever mode you're in.

Check your display manufacturer's site, AVS Forums, or search the model number.

Run the Windows HDR Calibration app first to find your display's clipping point — use that value as your peak brightness in RenoDX.

Why Is Cyberpunk's HDR Broken?

// Technical explanation from ShortFuse (RenoDX creator) ▾
CLICK TO EXPAND
  • SDR uses LUTs that clip instead of rolling off properly — they're all baked with ACES per-channel coloring and color tuning.
  • For HDR, CDPR took their SDR LUTs and stretched them to 1900 nits. But the SDR LUTs weren't all the same — some clipped at 100 nits, others at 325 nits, so the stretching is inconsistent.
  • After that they applied a second round of custom ACES HDR on top of those already-stretched LUTs. The correct approach would have been: Render → ACES-HDR. What they did was: Render → ACES-SDR-LUT-stretched-to-1900nits → CustomStretchedACESHDR.
  • On top of all that, their ACES would overshoot the user-selected peak by ~10%, and the more nits you set, the more per-channel color stretching occurred — meaning oversaturated, overcolored highlights.
  • The original RenoDX fix addressed the stretching from their custom ACES. The current Discord version targets the SDR look, knowing the HDR LUT is essentially broken ITM ACES-SDR.

Part 02
Configuring In-Game Settings & RenoDX
⚡ Windows HDR Calibration Required
Complete the Windows HDR Calibration before continuing. You need your display's clipping point.
Clipping Point vs. Peak Brightness:
• Peak Brightness — max brightness your display can output in current mode
• Clipping Point — where highlight detail starts being lost
• These should be the same value, but sometimes the clipping point is lower. Use the clipping point.
• True Black monitor owners: run calibration separately for each display mode.
💡 HDR10 PQ vs HDR10 scRGB

TLDR: No visual difference. Use HDR10 PQ for better ReShade compatibility.

If you want to use a ReShade preset like Creepy Ultra HDR, pick HDR10 PQ. scRGB is technically superior but you likely won't notice. Read Kaldaien's post for the technical breakdown.

8

Start the game and open the Settings menu.

9

Go to Video → HDR Settings. Make sure Cyberpunk's HDR is turned ON (recommend HDR10 PQ).

Maximum Brightness
Leave at default. Ignored by RenoDX — replaced by Peak Brightness in the addon.
Paper White (UI Brightness)
200 or adjust as desired. Controls UI brightness only when RenoDX is installed.
Tone-Mapping Midpoint
1.0 Critical for overall brightness. If the game looks too bright, double-check this value.
HDR10 PQ Saturation
MUST BE 0 Any non-zero value causes frame generation artifacts. Use RenoDX's saturation slider instead.
Screenshot: HDR Settings
10

In Graphics Settings, set Color Precision to High. Medium will result in a washed-out or gray image.

Screenshot: Color Precision setting
11

Open the ReShade interface with the Home key.

Don't have a Home key? Here's how to remap it.

12

Click the RenoDX tab at the top of the ReShade GUI. Set Peak Brightness to match your display's clipping point from the Windows HDR Calibration.

13

Game Brightness (paper white value) — set between 150–300 nits. Default recommendation is 200. May need to be adjusted depending on which LUT you're using.

14

Set SDR EOTF Emulation to UI/Menu only first. If everything still appears too bright, you can fully enable the setting.

15

Filmgrain — around 20 is a good starting point. This is from ShortFuse's ReShade shader repo.

If you're using Lilium's RCAS (a sharpening shader), set RenoDX filmgrain to 0 and add ShortFuse's filmgrain shader at the very end of your shader stack — you never want to sharpen filmgrain.

Double-check that Debug Graph is OFF (found at the very bottom of RenoDX under Debug). If it's on, you'll see a colored square behind the minimap in the upper right corner.

Troubleshooting

◌ Washed out / gray image
▼
  • Make sure RTXHDR is turned off globally (or at least for Cyberpunk 2077)
  • Verify HDR is on in both Windows and Cyberpunk 2077
  • Set Color Precision to High in graphics settings — RenoDX won't work on Medium
  • Make sure you're using the latest add-on version of ReShade
◌ Crushed black levels / shadows too dark
▼
  • Disable HDR10+ Gaming in the in-game video settings (only appears on HDR10+ capable displays)
  • Check that Tone-Mapping Midpoint is below 1.0 in the HDR settings
  • Some LUTs can cause this — try the vanilla LUT with RenoDX at default settings (keep Peak Brightness the same)
  • Disable any ReShade shaders, especially in HDR10 scRGB mode
  • Using Nova City 2? Darker interiors are expected behavior — the mod recommends flashlight/night vision mods
Screenshot: HDR10+ setting location
◌ Colored square behind minimap (upper right)
▼

This is the Debug Graph from RenoDX. Scroll to the very bottom of the RenoDX settings, find Debug, and turn off Debug Graph.

Screenshot: Debug Graph Off
◌ Banding / grid pattern in the sky
▼
  • Switch to Ray Reconstruction Preset D — slightly better for HDR in this specific case. Use DLSS Swapper for easy preset switching.
  • Note: Preset E is generally recommended over D (fewer artifacts, less boiling). Only switch to D if you're experiencing this specific banding/grid pattern issue.
  • When using Ray Reconstruction, it fully overrides the SR model — presets K, L, M, etc. will not apply.

Credit: squarto and Gn3xus for figuring this out.

◌ Frame generation artifacts when moving (especially around light sources)
▼

Set HDR10 PQ Saturation to 0 in the base game HDR settings. This is a confirmed fix.

Screenshot + Video: Before and after fix

Credit: Invictus for figuring this out and providing the video.

◌ Map is hard to read / washed out
▼

Known issue with this version of RenoDX. Affects the vanilla map and all map mods.

  • Partial fix: MapBegone mod by Cyanide disables the map ENV. Find it in the Cyberpunk 2077 Modding Server (search the pinned post in the linked channel).
  • Many people find the Realistic Map Mod (8K+) looks better with RenoDX — can be used with or without MapBegone.
◌ How do I adjust UI brightness?
▼

Adjust the Paper White value in the game's HDR settings. When RenoDX is installed, this setting only controls UI brightness and has no other effect.

Providing Info for Troubleshooting

1

Enable file name extensions in Windows File Explorer. This lets you see .ini, .txt, .log, and .exe extensions.

How to enable extensions in Windows 11
  1. Open File Explorer
  2. Click View in the top bar
  3. Hover over Show, then check File name extensions
2

Take screenshots using ReShade — these will produce HDR PNGs that can be uploaded to Discord.

  • SKIV is the only other accurate alternative
  • Nvidia overlay isn't always accurate; takes JXR screenshots that need conversion
  • Windows Snipping Tool and ShareX cannot capture HDR — they only tonemap to SDR
  • Do not photograph your screen with a phone unless the issue is display-specific
